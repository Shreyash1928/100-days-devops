# 🚀 Day 3 – Docker Networking

## 🎯 Objective

The goal of Day 3 was to understand how Docker containers communicate with each other using Docker Networks.

Unlike virtual machines, Docker containers communicate through virtual networks created and managed by the Docker Engine.

---

# Topics Covered

* Docker Networking
* Bridge Network
* Host Network
* None Network
* Custom Bridge Network
* Docker DNS
* Container Communication
* Docker Network Commands

---

# Practical Tasks

* Listed available Docker networks.
* Inspected the default bridge network.
* Created a custom Docker network.
* Connected multiple containers to the same network.
* Verified communication between containers.
* Connected and disconnected containers from networks.
* Removed unused Docker networks.

---

# Key Learnings

* Every container is attached to a network.
* Docker automatically provides DNS resolution inside custom bridge networks.
* Containers on the same network can communicate using container names.
* Port mapping is only required for communication outside Docker.
* Custom bridge networks are recommended for production deployments.
